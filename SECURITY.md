## Signaler une Vulnérabilité

Si vous découvrez une vulnérabilité de sécurité dans Flutter Booking, nous vous encourageons à nous la signaler de manière responsable.

### Comment signaler

1. **Ne pas** créer d'issue publique sur GitHub
2. Envoyer un email à l'équipe de sécurité avec les détails
3. Inclure autant d'informations que possible :
   - Description de la vulnérabilité
   - Étapes pour reproduire
   - Impact potentiel
   - Suggestions de correction si disponibles

### Processus de traitement

1. **Accusé de réception** : Nous accuserons réception de votre rapport dans les 48 heures
2. **Évaluation** : Notre équipe évaluera la vulnérabilité dans les 7 jours
3. **Correction** : Nous travaillerons sur un correctif selon la criticité
4. **Publication** : Une fois corrigée, nous publierons une mise à jour

### Délais de réponse

- **Critique** : Correction dans les 24-48 heures
- **Élevée** : Correction dans les 7 jours
- **Moyenne** : Correction dans les 30 jours
- **Faible** : Correction dans la prochaine version planifiée

## Bonnes Pratiques de Sécurité

### Pour les Développeurs

- Toujours valider les entrées utilisateur
- Utiliser HTTPS pour toutes les communications
- Garder les dépendances à jour
- Suivre les principes de moindre privilège
- Chiffrer les données sensibles

### Pour les Utilisateurs

- Utiliser des mots de passe forts
- Activer l'authentification à deux facteurs si disponible
- Maintenir l'application à jour
- Signaler tout comportement suspect

## Dépendances de Sécurité

Nous surveillons activement nos dépendances pour les vulnérabilités connues :

- Firebase SDK (authentification et base de données)
- Flutter framework
- Packages tiers via `pub.dev`
